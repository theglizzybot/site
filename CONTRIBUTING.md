# Contributing

Thanks for wanting to contribute! Please follow these steps to make collaboration smooth.

1. Fork the repository and create a feature branch:

```
git checkout -b feature/my-change
```

2. Make changes and include tests where applicable.

3. Commit with a clear message and open a Pull Request describing the change and rationale.

Guidelines
- Keep changes focused and small.
- Run the dev server locally and ensure no runtime errors.
- If your change affects configuration, update the wiki `Configuration.md` accordingly.

If you want, I can add a `CONTRIBUTING.md` at repository root and a simple PR template under `.github/PULL_REQUEST_TEMPLATE.md`.
